# Riftbound Card Crawler

A web scraping toolkit to extract all card data from the [Riftbound card gallery](https://riftbound.leagueoflegends.com/en-us/card-gallery/) and download card artwork. This is a personal project aims to both entertain my interest in the new Trading Card Game, and learn more about AI and website building.

---

## Purpose

This project automates two steps:

1. **Crawling** the Riftbound TCG card gallery to extract structured card data (name, ID, image URL, type, stats, etc.) into an Excel file
2. **Downloading** all card artwork images using the extracted URLs, saved by card ID

## Usage

### Step 1 â€” Crawl the card gallery

```bash
python riftbound_crawler.py
```

This will:

- Open a headless Chromium browser
- Navigate to the Riftbound card gallery
- Select "All" cards from the set filter
- Visit each card's detail page concurrently
- Save all card data to `riftbound_cards.xlsx`

**Options:**

| Flag | Description |
| ---- | ----------- |
| `--no-headless` | Show the browser window (useful for debugging) |
| `--concurrency 20` | Number of tabs to open simultaneously (default: `10`) |
| `--output my_file.xlsx` | Custom output filename |

---

### Step 2 â€” Download card images

```bash
python image_downloader.py
```

This will:

- Read `riftbound_cards.xlsx`
- Download every image from the `image` column
- Save each file to `images/{id}.jpg` (or `.png`, etc.)

---

## Output

### `riftbound_cards.xlsx`

| Column | Description |
| ------ | ----------- |
| `name` | Card name |
| `id` | Unique card ID |
| `image` | URL of the card artwork |
| `type` | Card type (e.g. Unit, Spell) |
| `region` | Card region / faction |
| `rarity` | Card rarity |
| `cost` | Mana cost |
| `power` | Attack power |
| `health` | Health points |
| `keywords` | Comma-separated list of keywords |
| `description` | Card ability text |

### `images/`

Card artwork saved as `{id}.jpg`, e.g. `RB-001.jpg`, `RB-002.png`, etc.

---

## Notes

- The crawler uses `asyncio.Semaphore` to limit concurrent browser tabs and avoid rate limiting
- The gallery page requires JavaScript to render, which is why Playwright is used over simpler HTTP libraries
- If selectors break due to a site update, run with `--no-headless` to inspect the DOM
